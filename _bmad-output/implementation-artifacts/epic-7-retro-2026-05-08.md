## Successes

1. **Two genuine production bugs found and fixed.** Story 7.4's `filterPayloadToDesiredKeys` resolved a systemic bug where 41 types could enter unnecessary write loops when Vault returned extra metadata fields. Story 7.5 fixed `PeriodicReconcilePredicate.Update()` reading a stale cached `ReconcileInterval` instead of the dynamic `SyncPeriod`. Both bugs were latent — only discovered because of thorough integration testing.

2. **Longest-carried debt item resolved.** The AC#4 extra-field handling issue, tracked since Epic 1 (5 epics ago), was finally completed in Story 7.4. The fix is elegant — a 6-line helper function applied consistently to all types.

3. **Story 7.0 refactoring paid immediate dividends.** Extracting shared test helpers first (1680 lines removed, 52% reduction) provided `waitForReconcileSuccess`, `SetupKVv2Stack`, and fixture constants reused in Stories 7.3 and 7.5.

4. **Story 7.1: perfect execution.** Zero debug failures, zero review findings across 54 test cases covering 27 types. Table-driven webhook validation is a solved pattern.

5. **Coverage gains in api/v1alpha1.** The unit-test-focused stories (7.1, 7.2) grew api/v1alpha1 coverage by 8.6 percentage points (16.9% → 25.5%), covering webhook validation and credential resolution paths that had zero prior coverage.

6. **Phase 1 test stabilization complete.** With 90 integration specs, comprehensive unit coverage for all 46 types' `IsEquivalentToDesiredState`, webhook validation, credential resolution, error paths, and drift detection — the project has a robust safety net for all future changes.

## Challenges

1. **Review findings tripled (4 → 15).** The jump correlates with epic novelty — Epic 7 introduced new testing patterns (unit tests, error paths, production code fixes, drift detection) versus Epic 6's repetitive integration test pattern. Findings were concentrated in assertions that proved *something happened* rather than *the right thing happened*.

2. **Story 7.4 scope was massive.** 45 files modified, 41 types fixed, 48 existing tests updated. Compilation errors from incorrect struct field names required grepping actual type definitions. Integration test timeout required longer await.

3. **Assertion rigor gap.** Stories 7.3–7.5 shared a pattern: initial implementations used weak assertions (non-empty strings, ContainSubstring) rather than exact match or negative proofs. All caught in review and fixed, but the pattern suggests story specs should be more explicit about proof requirements.

4. **PKI dual bug still unaddressed.** Carried through 6 consecutive epics (2–7). Confirmed non-blocking but represents accumulated minor debt.

---

## Key Insights

1. **"Hardening" epics find real production bugs.** The test safety net investment pays for itself — two genuine bugs found that would have caused operator users unnecessary Vault writes and broken runtime drift detection interval changes.

2. **Review finding rate tracks novelty and complexity, not quality regression.** Epic 6 (repetitive pattern, 4 findings) vs Epic 7 (novel patterns + production fixes, 15 findings) confirms this. High finding count in complex epics is expected and healthy.

3. **Refactoring stories placed first in an epic deliver immediate reuse.** Story 7.0's shared helpers were used in 2 subsequent stories within the same epic, amplifying the refactoring's value.

4. **The `filterPayloadToDesiredKeys` pattern is the canonical approach.** Should be documented in project-context.md so new types follow it automatically without re-discovering the problem.
